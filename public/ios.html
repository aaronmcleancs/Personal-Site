RepBook

Full Stack iOS Fitness Tracking Application with RESTful API and Enhanced Security.

Project Overview

RepBook is a comprehensive full-stack iOS application designed to revolutionize personal fitness tracking and workout planning. Developed using Objective-C, Swift, Node.js, and PostgreSQL, the application offers a seamless experience for users to monitor their fitness journey. Users can access a personalized dashboard to view real-time metrics, build custom workouts, and interact with an AI assistant that provides tailored fitness recommendations based on their exercise history.

The backend is powered by a robust RESTful API, ensuring secure data management and real-time synchronization between the client and server. User authentication is fortified using bcrypt for password hashing, and sensitive data is securely stored using Apple's Keychain on iOS devices. The application emphasizes modular design principles, employing a Network Manager utility to abstract and optimize network interactions, enhancing scalability and maintainability.

// Express.js route handler for user login
app.post('/login', async (req, res) => {
    try {
        const { email, username, password } = req.body;
        let query, values;
        if (email) {
            query = 'SELECT member_id, password, auth_key FROM members WHERE email = $1';
            values = [email];
        } else if (username) {
            query = 'SELECT member_id, password, auth_key FROM members WHERE username = $1';
            values = [username];
        } else {
            return res.status(400).send('Email or username is required');
        }

        const result = await pool.query(query, values);
        if (result.rows.length > 0) {
            const user = result.rows[0];
            const match = await bcrypt.compare(password, user.password);

            if (match) {
                res.json({ member_id: user.member_id, auth_key: user.auth_key });
            } else {
                res.status(401).send('Invalid credentials');
            }
        } else {
            res.status(401).send('Invalid credentials');
        }
    } catch (err) {
        console.error(`Error during login: ${err.message}`);
        res.status(500).send(err.message);
    }
});

// Swift implementation for secure storage using Keychain
class KeychainManager {
    static func save(_ data: Data, service: String, account: String) -> OSStatus {
        let query: [String: Any] = [
            kSecClass as String: kSecClassGenericPassword,
            kSecAttrService as String: service,
            kSecAttrAccount as String: account,
            kSecValueData as String: data
        ]
        SecItemDelete(query as CFDictionary)
        return SecItemAdd(query as CFDictionary, nil)
    }

    static func load(service: String, account: String) -> Data? {
        let query: [String: Any] = [
            kSecClass as String: kSecClassGenericPassword,
            kSecAttrService as String: service,
            kSecAttrAccount as String: account,
            kSecReturnData as String: kCFBooleanTrue!,
            kSecMatchLimit as String: kSecMatchLimitOne
        ]

        var item: CFTypeRef?
        let status = SecItemCopyMatching(query as CFDictionary, &item)
        guard status == noErr else { return nil }

        return (item as? Data)
    }
    
    static func delete(service: String, account: String) -> OSStatus {
        let query = [
            kSecClass as String: kSecClassGenericPassword,
            kSecAttrService as String: service,
            kSecAttrAccount as String: account
        ] as [String: Any]

        return SecItemDelete(query as CFDictionary)
    }

    static func loadAuthKey() -> String? {
        guard let authKeyData = load(service: "YourAppService", account: "authKey"),
              let authKey = String(data: authKeyData, encoding: .utf8) else {
            return nil
        }
        return authKey
    }
}

// Swift Network Manager implementation
static func createWorkout(memberId: Int, workoutName: String, exerciseIds: [Int], authKey: String, completion: @escaping (Result) -> Void) {
    guard let url = URL(string: "http://192.168.0.176:3000/createWorkout/\(memberId)") else {
        print("Invalid URL")
        completion(.failure(NetworkError.invalidURL))
        return
    }
    
    var request = URLRequest(url: url)
    request.httpMethod = "POST"
    request.addValue("application/json", forHTTPHeaderField: "Content-Type")
    request.addValue(authKey, forHTTPHeaderField: "Auth-Key")
    
    let body: [String: Any] = [
        "workoutName": workoutName,
        "exerciseIds": exerciseIds
    ]
    
    do {
        request.httpBody = try JSONSerialization.data(withJSONObject: body, options: [])
    } catch {
        print("Error encoding request body: \(error)")
        completion(.failure(error))
        return
    }
    
    URLSession.shared.dataTask(with: request) { data, response, error in
        if let error = error {
            print("Error creating workout: \(error.localizedDescription)")
            completion(.failure(error))
            return
        }
        
        guard let httpResponse = response as? HTTPURLResponse, httpResponse.statusCode == 200 else {
            print("Unexpected response status code")
            completion(.failure(NetworkError.serverError))
            return
        }
        
        completion(.success(()))
    }.resume()
}

Security Implementation

Security is a paramount concern in RepBook's architecture. The application implements multiple layers of protection to ensure data integrity and user privacy:

Password Security with Bcrypt

User passwords are secured using the bcrypt hashing algorithm, which provides several critical security advantages:

  • Slow Hash Computation: Bcrypt is deliberately designed to be computationally intensive, making brute-force attacks impractical by increasing the time required to hash each password.
  • Salt Integration: Each password is automatically combined with a unique salt before hashing, preventing rainbow table attacks and ensuring that identical passwords produce different hash values.
  • Adaptive Work Factor: The cost parameter can be adjusted over time to maintain security as computing power increases, ensuring long-term protection against evolving threats.

// Server-side implementation of password hashing with bcrypt
app.post('/signup', async (req, res) => {
    try {
        const { firstName, lastName, dateOfBirth, email, password, username, 
                heightCm, weightKg, gender, workoutFrequency } = req.body;
        
        // Validate required fields
        if (!firstName || !lastName || !dateOfBirth || !email || !password || 
            !username || !heightCm || !weightKg || !gender || !workoutFrequency) {
            return res.status(400).json({ error: 'Missing required fields' });
        }
        
        // Generate a strong salt and hash the password
        const saltRounds = 12;
        const hashedPassword = await bcrypt.hash(password, saltRounds);
        
        // Generate a unique authentication key
        const authKey = crypto.randomBytes(32).toString('hex');
        
        // Store user data with hashed password in database
        const result = await pool.query(
            'INSERT INTO members (first_name, last_name, date_of_birth, email, password, username, auth_key) VALUES ($1, $2, $3, $4, $5, $6, $7) RETURNING member_id',
            [firstName, lastName, dateOfBirth, email, hashedPassword, username, authKey]
        );
        
        // Return success response with member_id and auth_key
        res.status(201).json({ 
            member_id: result.rows[0].member_id, 
            auth_key: authKey 
        });
    } catch (err) {
        console.error(`Error during sign up: ${err.message}`);
        res.status(500).send(err.message);
    }
});

Apple Keychain Integration

On the iOS client, sensitive data like authentication tokens are stored using Apple's Keychain Services framework, which provides:

  • Encrypted Storage: Data is encrypted at rest using hardware-backed encryption on device.
  • Secure Access Control: Access to Keychain items can be restricted to specific applications and protected by biometric authentication if needed.
  • Secure Enclaves: On devices with Secure Enclaves, cryptographic operations are performed in isolated hardware, providing an additional security layer.

Authentication Token Management

The application implements a custom authentication token system:

  • Unique Token Generation: Each user receives a unique authentication key generated using cryptographically secure random bytes.
  • Token Verification: All protected API endpoints validate the Auth-Key header against the stored token for the user before processing requests.
  • Secure Storage: The token is stored securely in the Keychain on the client device and in the database on the server.

// iOS client implementation of saving auth token to Keychain
class AuthManager {
    static func saveAuthCredentials(memberId: Int, authKey: String) -> Bool {
        // Convert auth key to Data
        guard let authKeyData = authKey.data(using: .utf8) else {
            print("Error converting auth key to data")
            return false
        }
        
        // Save auth key to Keychain
        let authKeyStatus = KeychainManager.save(authKeyData, service: "YourAppService", account: "authKey")
        
        // Save member ID to Keychain
        let memberIdString = String(memberId)
        guard let memberIdData = memberIdString.data(using: .utf8) else {
            print("Error converting member ID to data")
            return false
        }
        let memberIdStatus = KeychainManager.save(memberIdData, service: "YourAppService", account: "memberId")
        
        return authKeyStatus == errSecSuccess && memberIdStatus == errSecSuccess
    }
    
    static func getAuthCredentials() -> (memberId: Int?, authKey: String?) {
        // Retrieve auth key from Keychain
        let authKey = KeychainManager.loadAuthKey()
        
        // Retrieve member ID from Keychain
        guard let memberIdData = KeychainManager.load(service: "YourAppService", account: "memberId"),
              let memberIdString = String(data: memberIdData, encoding: .utf8),
              let memberId = Int(memberIdString) else {
            return (nil, authKey)
        }
        
        return (memberId, authKey)
    }
    
    static func clearAuthCredentials() -> Bool {
        let authKeyStatus = KeychainManager.delete(service: "YourAppService", account: "authKey")
        let memberIdStatus = KeychainManager.delete(service: "YourAppService", account: "memberId")
        
        return authKeyStatus == errSecSuccess && memberIdStatus == errSecSuccess
    }
}

Architecture and Implementation

The architecture of RepBook is grounded in modern software engineering principles, emphasizing modularity, scalability, and security. The system is built on a client-server model with a clear separation of concerns:

Backend Architecture

  • RESTful API: The backend employs a RESTful API design, allowing stateless communication between the client and server. This facilitates easier scaling and maintenance.
  • Express.js Framework: Built on Node.js using Express.js to handle HTTP requests and routing in a lightweight and efficient manner.
  • PostgreSQL Database: Implements a relational database with normalized schema design for efficient data storage and retrieval.
  • Connection Pooling: Utilizes connection pooling to efficiently manage database connections and improve performance under load.
  • Middleware Architecture: Implements middleware for cross-cutting concerns like logging, CORS handling, and authentication verification.

// Express.js middleware for authentication verification
const authMiddleware = async (req, res, next) => {
    try {
        const authKey = req.headers['auth-key'];
        const memberId = req.params.memberId || req.body.memberId;
        
        if (!authKey || !memberId) {
            return res.status(401).send('Authentication required');
        }
        
        const result = await pool.query(
            'SELECT auth_key FROM members WHERE member_id = $1',
            [memberId]
        );
        
        if (result.rows.length === 0) {
            return res.status(404).send('Member not found');
        }
        
        if (result.rows[0].auth_key !== authKey) {
            return res.status(401).send('Invalid authentication');
        }
        
        next();
    } catch (err) {
        console.error(`Authentication error: ${err.message}`);
        res.status(500).send('Internal server error');
    }
};

iOS Client Architecture

  • Model-View-Controller (MVC) Pattern: Organizes code into Models (data structures), Views (UI components), and Controllers (business logic).
  • Network Abstraction Layer: Implements a dedicated NetworkManager class to handle all API communication, decoupling networking from business logic.
  • Security Services: Uses the KeychainManager to securely store and retrieve sensitive credentials.
  • Data Models: Defines clear data models with Codable conformance for seamless JSON serialization/deserialization.
  • Error Handling: Implements robust error handling with custom error types and user-friendly error messages.

Data Flow Architecture

The application implements a clear data flow pattern:

  1. Request Initiation: User interactions trigger view controllers to request data or actions.
  2. Network Layer: The NetworkManager class constructs appropriate API requests with authentication headers.
  3. Server Processing: The Express.js server validates requests, processes them, and interacts with the PostgreSQL database.
  4. Response Handling: Responses are parsed into Swift model objects and used to update the UI.
  5. Error Handling: Errors at any stage are caught, logged, and presented to the user when appropriate.

Computer Science Concepts Explored

  • Cryptography: Implementation of secure hashing algorithms (bcrypt) and random token generation for authentication.
  • Database Normalization: Proper schema design with normalized tables to minimize redundancy and improve data integrity.
  • REST Architecture: Implementation of RESTful API principles for stateless client-server communication.
  • Object-Oriented Design: Use of classes, inheritance, and encapsulation to create modular, maintainable code.
  • Asynchronous Programming: Implementation of asynchronous programming paradigms to ensure non-blocking operations and efficient handling of API requests.
